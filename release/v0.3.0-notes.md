# CS2FOW v0.3.0

CS2FOW 0.3.0 supports CS2 build `24304127` and the pinned Metamod:Source/HL2SDK interface.

## What changed

- Replaced the fifteen hand-tuned body points and eight AABB corners with Valve's nineteen live animated hitbox capsules.
- Evaluates the complete projected capsule volumes through a bounded 32x32 CPU depth buffer while retaining the separate weapon-muzzle point.
- Added a configurable one-to-four-thread visibility pool, with two workers by default, plus fair work rotation, a safe visibility-ray prepass, and reveal-hold reuse.
- Expanded and compacted verified occluder proofs. The final 32-player live benchmark cut moving wall time from 10.33 ms to 7.79 ms and rasterized triangles from 160,521 to 80,557 without changing visibility decisions.
- Expanded `cs2fow_status` with wall and aggregate CPU timing, recent p95/p99 latency, probe, hold, cache, proof, and compaction counters.
- Preserved the existing viewing origins, smoke occlusion, HE smoke clearing, teammate rules, and `CheckTransmit` behavior.
- Invalid capsule capture, uncertain sub-pixel coverage, worker cancellation, or exhaustion of the 75 ms budget still fails open and leaves the player visible.

## Installation

1. Update the CS2 server and Metamod:Source.
2. Extract the appropriate Windows or Linux Release asset into the server's `game/csgo` folder without rearranging it.
3. Load a map and run `cs2fow_status`.

CS2FOW intentionally remains inactive when the loaded CS2 server binary does not match the verified gamedata fingerprint.

## Validation

- Windows and Steam Runtime Linux builds pass the complete runtime test suite.
- The deterministic proof-compaction test preserves blocked/visible decisions.
- The Windows 32-player A/B benchmark reduces rasterized triangles by 29% and mixed-workload time by approximately 5% locally.
- The live 32-player server benchmark reduces moving wall time by 24.6%, aggregate worker CPU time by 24.2%, and snapshot age by 30.3%, with no budget exhaustion, cache saturation, uncached blocked result, or capsule-capture failure.

Compiled ZIPs are distributed only as GitHub Release assets and are not stored or linked from the README.
