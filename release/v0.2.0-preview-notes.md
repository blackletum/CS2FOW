# CS2FOW v0.2.0-preview

This preview reorganizes CS2FOW so the runtime and safety rules can be followed without reading one giant plugin file. It also replaces the old map-bake format and removes CS2FOW's use of unverified internal transmit lists.

## Main changes

- The runtime is separated into top-level plugin control, live game-state capture, background visibility work, CheckTransmit filtering, and automatic baking.
- CheckTransmit clears only the verified primary entity-send list. Full-update snapshots are left untouched.
- Once a visual group is hidden, it waits for CS2's next natural full update before reappearing. CS2FOW never forces that refresh.
- `cs2fow_entity` now reports fixed-size, aggregated evidence for bits that were genuinely set before CS2FOW cleared them.
- BVH8 version 3 validates the complete file layout and payload CRC, verifies a temporary write, and atomically replaces the old bake only after that verification succeeds.
- The C++ baker now owns all Valve package discovery/extraction and supports `--list-maps --vpk <file>`.
- The target samples include the existing custom body points and axis-aligned bounding box corners plus held-weapon muzzle handling.
- The README starts with installation and operation. `docs/CODE_TOUR.md` explains the complete runtime in plain language.

## Installing

1. Install Metamod:Source.
2. Extract the Windows or Linux core ZIP into the server's `game/csgo` folder.
3. Optionally extract the official-map ZIP into the same folder.
4. Load a map and run `cs2fow_status`.

Version 2 and older `.bvh8` files are intentionally rejected. When the packaged tools can run and the map-data folder is writable, CS2FOW automatically rebuilds a missing or old bake. It remains fail open while doing so.

## What the checks prove

The automated suite covers map/VPK parsing, BVH8 validation and replacement, open/blocked rays, visibility sampling and worker behavior, lifecycle/transmit helpers, debug evidence, package contents, checksums, editor-point agreement, and Windows/Linux builds.

These checks do not reproduce a real CS2 transmit list, prove live-server crash safety, or explain every engine-side cause of `CopyExistingEntity: missing client entity`. Live-server packet testing remains a separate validation step.

This is a prepared release candidate only. No tag, GitHub release, upload, or Bake Service deployment has been created.
